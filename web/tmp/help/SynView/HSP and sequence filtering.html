How can I filter my HSPs from the analysis?
 
SynView provides two ways to filter HSPs from the results.  Our preferred way involves our unique (but not proprietary) HSP filter.  This method adds 13-18 nucleotides of fixed sequence (aka the spike sequence) on the the ends of both sequences for each blast analysis.  The code (written by Brian C. Thomas, unpublished) avoids boundary difficulties where the authentic sequence meets the spike sequence by adding a short linker sequence that is not identical.  In the above example, this HSP filter is turned on with a 15/15 nucleotide spike sequence.  HSPs with an e-value greater than the e-value obtained by the HSP containing this spike sequence will not be displayed in the results.  We have found that dropping the filter below a 15/15 exact match in plant genomic sequence comparisons generates false positives (noise) when searching for conserved non-coding sequences (data not shown).  Overall, this HSP filter provides an excellent way to remove a lot of the spurious, non-informative blast HSPs from the results visualization.  Also, it is important to remember that these HSPs are not removed from the raw blast results and can be seen by downloading blast results file (available in the SynView results).
 
It is also possible to remove HSPs above a significance level by changing the expectation limit setting for bl2seq itself.  This will result in HSPs above an e-value to not be reported by bl2seq.  If you only want to use this filter, turn the "HSP filter" off.

How do I mask coding or non-coding sequence from my sequences?
 
SynView again makes this easy!  All you need to do is select the appropriate radio button to turn on or off sequence masking for either coding sequence (exons) or non-coding sequence (everything else).  One thing that you need to keep in mind that SynView can only mask coding sequences if it knows where the coding regions are in your sequence.  If you use the CoGe database sequence retrieval, genbank internet sequence retrieval, or directly submit a sequence in genbank format, then SynView can determine the coding sequences if they are annotated as such.  However, SynView cannot do the same for Fasta format sequences as there is not a standard convention for annotated sequences in that format.
