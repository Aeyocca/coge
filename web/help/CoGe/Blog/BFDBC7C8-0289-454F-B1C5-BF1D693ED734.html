Blastz implemented for SynView

Blastz has been added as a Blast program option under the currently inaptly named menu "Bl2Seq Parameters". Blastz is a variant of blast that has been optimized for whole genome alignments. It does its best to find large blocks of conserved sequence. Thus, if you are scanning genomic regions for synteny, blastz does a better job of producing large HSPs making it easier to read the result graphics. As you can garner from the image above, the blast parameters don't work for blastz (yet). This is because blastz uses completely different notations than bl2seq for parameter specifications and the scores used (sequence matrix, gap opening, etc) have no relation to the scores used in bl2seq. Fortunately, the default parameters for blastz do a great job for the cases I've examined. However, when I get the time, I will add support for blastz parameters. If you are curious to see the difference, below are two images from SynView using bl2seq (blastn) and blastz. Also, these images preview new methods for drawing gene models that was coded by Josh Kane.
