How can I filter my HSPs or mask parts of my sequence from the analysis?
 
SynView provides a novel HSP filter based on a user specified length of an exact blast match as well as an easy way to mask coding or non-coding sequences from your analysis.  The HSP filter provides an excellent way to remove a lot of the spurious, non-informative blast HSPs from the results visualization.  Masking coding sequences helps in the identification of conserved non-coding sequences, and masking non-coding sequences helps in the identification of synteny or gene duplications.

How do I filter HSPs based on length of a sequence match?
 
SynView provides a novel HSP filter that will remove any HSP whose e-value is above the e-value of an exact match HSP of a user specified length.  This is achieved by attaching a "spike" sequence to the end of the submitted sequences (of the length specified by you, the user -- see the above figure).  Then after the bl2seq analysis is run, SynView searches through the HSPs for the one that contains the spike sequence, determining its associated e-value, then only will display HSPs with an e-value less than that.  Use the "On/Off" radio buttons next to the "HSP Filter" option to turn this filter on or off, and select the length of the spike sequence from the radio buttons listed below "Only show matches of equal to or greater than:".

How do I mask coding or non-coding sequence from my sequences?
 
SynView again makes this easy!  All you need to do is select the appropriate radio button to turn on or off sequence masking for either coding sequence (exons) or non-coding sequence (everything else).  One thing that you need to keep in mind that SynView can only mask coding sequences if it knows where the coding regions are in your sequence.  If you use the CoGe database sequence retrieval, genbank internet sequence retrieval, or directly submit a sequence in genbank format, then SynView can determine the coding sequences if they are annotated as such.  However, SynView cannot do the same for Fasta format sequences as there is not a standard convention for annotated sequences in that format.
